# freee キャッシュフロー予測ダッシュボード ― 完全手順書

このドキュメント1本で、ゼロから本番運用まで完結します。所要時間：約45分（freee登録手順を含む）。


## Phase 0 ― 事前準備

### 0-1. チェックリスト

| 項目 | 必要 | 確認方法 |
|------|------|---------|
| Python 3.10 以上 | 必須 | ターミナルで `python --version` |
| ブラウザ（Chrome / Edge / Safari 最新版） | 必須 | バージョン確認 |
| freee会計（法人プラン） | 実データ利用時 | freee管理画面 |
| freee人事労務 | 給与データ利用時 | freee管理画面 |
| freee管理者権限（アカウント） | 実データ利用時 | freee管理画面 → ユーザー設定 |

### 0-2. Pythonが入っていない場合

- **Windows**：<https://www.python.org/downloads/> からダウンロード。インストール時に **「Add Python to PATH」にチェック** を必ず入れる。
- **macOS**：Homebrew 利用なら `brew install python@3.12`、なければ上記公式サイトから。

確認：
```bash
python --version
# または
python3 --version
```
`Python 3.10` 以上の表示ならOK。

## Phase 1 ― アプリのインストール

### 1-1. ターミナルを開く

- **Windows**：スタートメニュー → 「PowerShell」と入力 → 起動
- **macOS**：Spotlight（⌘+Space）→ 「ターミナル」と入力 → 起動

### 1-2. フォルダに移動

```bash
cd "C:\Users\yugoh\Documents\Claude\Projects\会計及び労務\freee_cashflow"
```

macOSの場合は該当パスに読み替えてください。

### 1-3. 仮想環境を作る（推奨）

```bash
python -m venv .venv
```

### 1-4. 仮想環境を有効化

**Windows (PowerShell)**:
```powershell
.venv\Scripts\Activate.ps1
```
※ 「スクリプトの実行が無効」と出た場合：
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```
を一度だけ実行してから再度有効化。

**macOS / Linux**:
```bash
source .venv/bin/activate
```

ターミナルの行頭に `(.venv)` が付けば成功。

### 1-5. 依存ライブラリをインストール

```bash
pip install -r requirements.txt
```

5〜30秒で完了。エラーが出なければOK。

### 1-6. 環境変数ファイルを用意

**Windows**:
```powershell
copy .env.example .env
```

**macOS / Linux**:
```bash
cp .env.example .env
```

この時点で `.env` の中身は `MOCK_MODE=true` のままです。**変更不要**で次に進みます。

## Phase 2 ― Mockモードで動作確認

freee登録の前に、まずアプリの全機能を触ってみていただきます。

### 2-1. サーバー起動

```bash
uvicorn app.main:app --reload --port 8765
```

成功すると以下のような表示が出ます：
```
INFO:     Uvicorn running on http://127.0.0.1:8765 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

### 2-2. ブラウザで開く

<http://localhost:8765> を開いてください。

### 2-3. ダッシュボードを触る

最初に表示される画面で、以下を確認できます：

| エリア | 内容 |
|--------|------|
| 画面上部の黄色いバッジ「Mockモード」 | サンプルデータで動作中 |
| 4枚のKPIカード | 期首残高、12ヶ月後残高、最低残高月、純CF |
| 残高推移グラフ | 楽観・中立・悲観の3シナリオ折れ線 |
| CF予測表タブ | 月次×12ヶ月の詳細CF（シナリオ切替可） |
| サイト設定タブ | 売掛/買掛サイト、期首残高設定 |
| 借入金タブ | サンプル借入2本（運転資金・設備資金）入り |
| 手動調整タブ | 賞与・税金・設備投資の月別入力 |
| シナリオ詳細タブ | 3シナリオの比較サマリー |

### 2-4. 試しに値を変更してみる

例：「サイト設定」タブで売上回収サイトを `1` → `2` に変更 → 「設定を保存」 → グラフ・KPIが即座に再計算されます。

### 2-5. サーバー停止

ターミナルで `Ctrl + C` を押すと停止します。

ここまでで動作が確認できれば、アプリ本体に問題はありません。次のPhaseでfreee連携に進みます。

## Phase 3 ― freee Developersでアプリ登録

### 3-1. freee Developersにアクセス

ブラウザで <https://app.secure.freee.co.jp/developers/applications> を開きます。

普段freeeにログインされているアカウントと同じアカウントでログインしてください（**管理者権限のあるアカウント** を推奨）。

初回アクセスの場合、「freee for Developers 利用規約」に同意します。

### 3-2. 新規アプリ作成

1. 画面右上の **「新規追加」** をクリック。
2. 以下を入力：

   | 項目 | 入力値 |
   |------|--------|
   | **アプリ名** | `社内キャッシュフロー予測`（任意） |
   | **概要** | `社内のCF予測ツール`（任意） |
   | **アプリタイプ** | **`プライベートアプリ`** を選択（自社内利用） |

3. **「作成」** をクリック。

### 3-3. コールバックURLを設定

作成したアプリの詳細画面に遷移します。「基本設定」タブで以下を入力：

| 項目 | 入力値 |
|------|--------|
| **コールバックURL** | `http://localhost:8765/oauth/callback` |

入力後、**「下書き保存」** をクリック。

### 3-4. 権限（スコープ）を設定

同じ画面の「権限設定」タブを開き、以下を **「読み取り」のみ** 有効化します。書き込み権限は不要です。

**freee会計**
- ☑ 事業所
- ☑ 取引
- ☑ 請求書
- ☑ 口座
- ☑ 取引先
- ☑ 勘定科目
- ☐ 経費精算（任意）

**freee人事労務**
- ☑ 事業所
- ☑ 従業員
- ☑ 給与明細

**「下書き保存」** をクリック。

### 3-5. Client ID / Client Secret を取得

「基本設定」タブに戻り、画面に表示されている以下2つの値をメモ帳などにコピー：

- **Client ID**（例：`a1b2c3d4e5f6...`）
- **Client Secret**（例：`x9y8z7w6v5u4...`）

⚠️ Client Secretは秘密情報です。他人に教えない・公開リポジトリに上げないでください。

## Phase 4 ― .envに認証情報を設定

### 4-1. APP_SECRET_KEY を生成

ターミナルで以下を実行：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

64桁の英数字が表示されます。これをコピーしてください。

### 4-2. .env を編集

`freee_cashflow/.env` をテキストエディタ（メモ帳、VS Code等）で開き、以下のように書き換えます：

```env
MOCK_MODE=false

FREEE_CLIENT_ID=（Phase 3-5でコピーしたClient ID）
FREEE_CLIENT_SECRET=（Phase 3-5でコピーしたClient Secret）
FREEE_REDIRECT_URI=http://localhost:8765/oauth/callback

APP_SECRET_KEY=（Phase 4-1で生成した64桁文字列）

APP_PORT=8765
```

保存して閉じます。

## Phase 5 ― freeeと連携して実データ取得

### 5-1. サーバー再起動

Phase 2で起動したサーバーが動いている場合は `Ctrl + C` で停止し、改めて：

```bash
uvicorn app.main:app --reload --port 8765
```

### 5-2. ブラウザでダッシュボードを開く

<http://localhost:8765>

画面上部のバッジが **「未連携」** になっているのを確認。

### 5-3. 「freeeと連携」ボタンを押す

画面右上の **「freeeと連携」** ボタンをクリック。

### 5-4. freeeで認可

ブラウザがfreeeのログイン画面に遷移します：

1. freeeのIDとパスワードでログイン（既にログイン済みなら自動でスキップ）。
2. 「対象事業所を選択してください」画面で、対象の事業所を選択。
3. 「アプリ連携を開始しますか？」画面で **「許可する」** をクリック。

### 5-5. 自動で戻る

「✅ freeeとの連携が完了しました」と表示され、3秒後にダッシュボードに自動遷移します。

バッジが **「● freee連携中」（緑）** になり、自社のデータでKPIとグラフが再描画されます。

### 5-6. データ取得に失敗する場合

画面上部に黄色の警告バナーが出てMockデータに自動フォールバックされた場合は、**Phase 9（トラブルシューティング）** をご確認ください。

## Phase 6 ― 自社向けにカスタマイズ

### 6-1. サイト設定の調整

「サイト設定」タブで、自社の実態に合わせて変更：

| 項目 | 例 | 説明 |
|------|----|------|
| 売上回収サイト | `1` | 月末締め翌月末入金なら `1` |
| 仕入・経費支払サイト | `2` | 月末締め翌々月末支払なら `2` |
| 給与支払サイト | `0` | 当月末締め当月支払なら `0`、翌月支払なら `1` |
| 期首現預金 | 空欄推奨 | 空欄ならfreeeの口座残高合計を自動利用 |

「設定を保存」 → 即座にCF予測が再計算されます。

### 6-2. シナリオパラメータの調整

同じく「サイト設定」タブの下部「シナリオパラメータ」表：

| シナリオ | 売上倍率の目安 | 原価倍率の目安 |
|---------|---------------|---------------|
| 楽観 | 1.10〜1.20 | 0.90〜0.95 |
| 中立 | 1.00 | 1.00 |
| 悲観 | 0.80〜0.90 | 1.05〜1.10 |

業界・経営環境に応じて自由に設定可能。

### 6-3. 借入金の登録

「借入金」タブで、サンプルとして入っている2本を削除し、実際の借入を登録：

| 項目 | 入力例 |
|------|-------|
| 借入名 | `〇〇銀行 短期運転資金` |
| 残高（円） | `25000000` |
| 年利 | `0.018`（1.8%なら） |
| 残月数 | `48`（残り4年なら） |
| 返済方式 | `元金均等` または `元利均等` |

「借入情報を保存」 → 月次の元金返済・利息支払が自動で財務CF・営業CFに反映されます。

### 6-4. 手動調整の登録

freee APIでは取れない、あるいは予測しづらい項目を月別に入力：

| 対象月 | 区分 | 項目 | 金額 |
|--------|------|------|------|
| 2026-07 | 営業 | 夏季賞与 | `-5000000` |
| 2026-09 | 投資 | 設備購入 | `-2500000` |
| 2026-11 | 営業 | 法人税中間納付 | `-3500000` |
| 2026-12 | 営業 | 冬季賞与 | `-5500000` |
| 2027-03 | 営業 | 確定申告 | `-4000000` |

「調整を保存」 → 該当月のCFに即時反映。

## 日常運用

### A. 毎朝の確認

```bash
cd freee_cashflow
source .venv/bin/activate   # Windows: .venv\Scripts\Activate.ps1
uvicorn app.main:app --port 8765
```

ブラウザで <http://localhost:8765> を開き、右上の **↻（リロード）** ボタンを押すとfreeeから最新データを再取得します。

### B. ショートカット起動を作る（任意）

毎回コマンドを打つのが面倒な場合：

**Windows** — `freee_cashflow/start.bat` を作成：
```batch
@echo off
cd /d "%~dp0"
call .venv\Scripts\activate.bat
start "" http://localhost:8765
uvicorn app.main:app --port 8765
```
ダブルクリックで起動できます。

**macOS** — `freee_cashflow/start.command` を作成：
```bash
#!/bin/bash
cd "$(dirname "$0")"
source .venv/bin/activate
open http://localhost:8765
uvicorn app.main:app --port 8765
```
ターミナルで `chmod +x start.command` を一度実行。以降ダブルクリックで起動。

### C. アクセストークンの自動更新

アクセストークンは6時間で期限切れになりますが、アプリが自動的にリフレッシュトークン（90日有効）で更新します。**Hikitaさまの操作は不要** です。

90日経過後は再度「freeeと連携」ボタンを押して再認可してください。

## Phase 9 ― トラブルシューティング

### Q1. `port 8765 is already in use` と出る
別のプロセスが同じポートを使用中です。`--port 8800` のように別ポートを指定し、`.env` の `FREEE_REDIRECT_URI` と freee Developers の **コールバックURL** も `http://localhost:8800/oauth/callback` に揃えてください。

### Q2. `Invalid grant` エラーで連携できない
リフレッシュトークン期限切れ（90日）か、既に使用済みです。
```bash
# data/tokens.json を削除して再連携
rm data/tokens.json   # Windowsは: del data\tokens.json
```
サーバー再起動後、改めて「freeeと連携」ボタンを押してください。

### Q3. 給与データが取得できない（給与支払が0円のまま）
freee人事労務の **管理者権限のあるアカウント** で連携してください。一般従業員権限では給与明細APIが叩けません。
代替策として「手動調整」タブで月別に給与額を入力する運用も可能です。

### Q4. ダッシュボードを開いても「読込中...」のまま
ブラウザのデベロッパーツール（F12）→ Console タブでエラーを確認。多くの場合はネットワーク（CDN）への接続失敗です。社内プロキシ環境の場合、Chart.jsのCDN（`cdn.jsdelivr.net`）への接続を許可してください。

### Q5. 数値が直感と合わない
- **入金が思ったより遅い** → 「サイト設定」の売上回収サイトが大きすぎないか確認
- **6月・12月の支払が大きい** → 「手動調整」に賞与を入れているか確認（Mockモードでは自動で乗っている）
- **借入残高が増えていく** → 借入の「残月数」が小さすぎる、または「返済方式」が想定と違う

### Q6. 完全にリセットしたい
```bash
rm data/tokens.json data/settings.json data/adjustments.json data/loans.json
# Windows: del data\*.json
```
次回起動時にすべて初期値からスタートします。

### Q7. 複数事業所がある場合
現バージョンは取得した先頭の事業所を対象にしています。事業所選択UIが必要な場合はお申し付けください、機能追加いたします。
